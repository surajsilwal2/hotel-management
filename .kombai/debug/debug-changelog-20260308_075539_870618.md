## Changes Made

### 1. Fixed `app/(guest)/rooms/page.tsx` — image always showed placeholder
- **File:** `app/(guest)/rooms/page.tsx`
- **Change 1:** Added `import Image from "next/image"`
- **Change 2:** Replaced the hardcoded gradient+icon placeholder with a conditional that renders `<Image fill>` when `room.images?.[0]` exists, falling back to the icon placeholder otherwise. - **Root cause:** The original code always rendered the `<BedDouble>` icon regardless of whether the room had images — it never checked `room.images`.
- **Revert:** Remove `Image` import; replace the conditional block with the original static gradient div.

### 2. Updated `prisma/seed.ts` — rooms had no images
- **File:** `prisma/seed.ts`
- **Change:** Added `ROOM_IMAGES` map with 1–3 real Unsplash photo URLs per room. Updated each `upsert` to include `images` in both `create` and `update` so re-running the seed patches existing rooms in the database.
- **Root cause:** Original seed had `images` field omitted, so Prisma defaulted to `[]` — rooms were always imageless.
- **Revert:** Remove `ROOM_IMAGES` map; set all upserts back to `update: {}` and remove `images:` from `create` blocks.
