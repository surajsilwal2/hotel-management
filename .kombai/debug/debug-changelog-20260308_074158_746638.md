### 3. Updated `app/globals.css`
- **File:** `app/globals.css`
- **Change:** Added `@theme { --font-sans: var(--font-inter), Inter, sans-serif; }` block to wire up the Inter font via CSS variable (the v3 tailwind.config.ts `fontFamily` equivalent in v4).
- **Note:** `@import "uploadthing/tw/v4"` was already correct ‚Äî `uploadthing/tw/v4.css` exists and uses `@custom-variant` (Tailwind v4-only). No change to this line.
- **Revert:** Remove the `@theme {}` block

### 4. Deleted `tailwind.config.ts`
- **File:** `tailwind.config.ts` (DELETED)
- **Change:** Tailwind v4 no longer uses a JS config file. Theme is configured via `@theme {}` in CSS. The `content` array is also not needed ‚Äî v4 auto-scans files.
- **Revert:** Re-create `tailwind.config.ts` with original content:
  ```ts
  import type { Config } from "tailwindcss";
  const config: Config = {
    content: [
      "./pages/**/*.{js,ts,jsx,tsx,mdx}",
      "./components/**/*.{js,ts,jsx,tsx,mdx}",
      "./app/**/*.{js,ts,jsx,tsx,mdx}",
    ],
    theme: {
      extend: {
        fontFamily: {
          sans: ["var(--font-inter)", "Inter", "sans-serif"],
        },
      },
    },
    plugins: [],
  };
  export default config;
  ```

### 5. Fixed `app/layout.tsx` ‚Äî Inter font variable setup
- **File:** `app/layout.tsx`
- **Change 1:** `Inter({ subsets: ["latin"] })` ‚Üí `Inter({ subsets: ["latin"], variable: "--font-inter" })`
- **Change 2:** `<body className={inter.className}>` ‚Üí `<body className={inter.variable}>`
- **Reason:** The old code used `inter.className` which directly applied the font but never created the `--font-inter` CSS variable referenced in the Tailwind theme. Now `inter.variable` sets `--font-inter`, which `@theme { --font-sans: var(--font-inter) }` picks up, and Tailwind v4 preflight applies `font-family: var(--font-sans)` to the document.
- **Revert:** Revert both changes above

### 6. Fixed `components/AvatarUploader.tsx` ‚Äî missing `relative` on Image fill container
- **File:** `components/AvatarUploader.tsx`
- **Change:** Added `relative` class to the avatar `<div>` that wraps `<Image fill>`
- **Reason:** `next/image` with the `fill` prop requires the parent element to have `position: relative` (or absolute/fixed). Without it the image is invisible and Next.js logs a console error.
- **Revert:** Remove `relative` from the class string

### 7. Fixed `next.config.mjs` ‚Äî missing UploadThing image domains
- **File:** `next.config.mjs`
- **Change:** Added `utfs.io` and `ufs.sh` to `images.remotePatterns`
- **Reason:** Room images and avatars are stored on UploadThing servers which use these domains. `next/image` blocks external domains not listed in `remotePatterns`, causing broken images in the admin rooms page and guest profile.
- **Revert:** Remove the two added `remotePatterns` entries
